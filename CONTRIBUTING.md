# 貢献ガイドライン

このプロジェクトへの貢献に興味を持っていただきありがとうございます。

## 開発環境のセットアップ

1. Rustの最新安定版をインストールしてください
2. リポジトリをフォークしてクローンしてください
3. 依存関係をインストールしてください：`cargo build`

## コーディング規約

### スタイルガイド

- Rustの標準的なコーディング規約に従ってください
- すべてのコードは`cargo fmt`でフォーマットしてください
- `cargo clippy`で警告が出ないようにしてください

### コミットメッセージ

明確で説明的なコミットメッセージを書いてください：

```
タイプ: 簡潔な説明

詳細な説明（必要に応じて）

関連するIssue番号（該当する場合）
```

**タイプの例：**
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメントのみの変更
- `style`: コードの意味に影響しない変更（空白、フォーマットなど）
- `refactor`: バグ修正や機能追加ではないコード変更
- `test`: テストの追加や修正
- `chore`: ビルドプロセスやツールの変更

## プルリクエストのプロセス

1. 最新のmainブランチから新しいブランチを作成してください
2. 変更を実装してください
3. テストを追加/更新してください
4. すべてのテストが通ることを確認してください：`cargo test`
5. コードをフォーマットしてください：`cargo fmt`
6. Clippyの警告を確認してください：`cargo clippy`
7. 変更内容を説明する明確なタイトルと説明でプルリクエストを作成してください

## テスト

- すべての新機能にはテストを含めてください
- バグ修正には、そのバグを再現するテストを含めてください
- テストは`tests/`ディレクトリまたは各モジュール内に配置してください

## ドキュメント

- 公開APIにはドキュメントコメントを追加してください
- 複雑なロジックにはコメントで説明を追加してください
- 必要に応じてREADME.mdを更新してください

## 質問やヘルプ

質問がある場合は、Issueを作成するか、既存のIssueにコメントしてください。
